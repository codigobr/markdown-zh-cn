基本Markdown语法

在本文中，您将学习构成Markdown语言基础的元素。

  1. Markdown
  3. 基本Markdown语法

概述

尽管有一些变化，但是大多数应用程序都支持Markdown的基本元素。这些元素对于您格式化文档至关重要。但是请不要误会，仅使用Markdown的基础知识，就可以为Web创建精心设计的站点。

注意：只要有可能，当从一个Markdown处理器到另一个Markdown处理器存在语法差异时，这些都会在本文中指出。

换行：最佳做法

尽管可以与大多数应用程序兼容，但使用空格创建换行符可能会导致格式问题。

以下是一些要点：

  • 行后的空白很难注意到。
  • 由于习惯或粗心大意，人们可以在行尾添加空格。
  • 某些文本编辑器（例如Sublime Text）具有在保存文档时从行中删除行前和行后空格的设置。

Uma alternativa viável para evitar qualquer problema com quebra de linhas, basta usar a tag HTML <br> diretamente nos seus documentos.

注意：有趣的是，大多数Markdown应用程序也与HTML中的tags兼容，也就是说，您可以在同一文档中将Markdown格式与HTML格式合并。
注意：在某些应用程序中，只需按Enter键即可创建换行符

不建议按Enter键或使用反斜杠\（尽管有效），因为兼容性取决于Markdown程序。

重点

类似于Microsoft Word等WYSIWYG编辑器，Markdown还提供了格式设置来强调文本。

最受欢迎的强调格式包括粗体和斜体，也可以使用Markdown语法来应用。

在Markdown中加粗

要在Markdown中设置粗体文本格式，请在内容之前放置两个星号**，在内容之后放置两个星号**。

例子

  Markdown

**This part of the text** will be bold.
自己尝试 

  HTML

<strong>This part of the text</strong> will be bold.

  结果

This part of the text will be bold.

  Markdown

__This part of the text__ will be bold.
自己尝试 

  HTML

<strong>This part of the text</strong> will be bold.

  结果

This part of the text will be bold.

  Markdown

Markdown**Is**Easy**To**Learn
自己尝试 

  HTML

Markdown<strong>Is</strong>Easy<strong>To</strong>Learn

  结果

MarkdownIsEasyToLearn
NOTE：如以上示例所示，您可以在内容还格式化Markdown之前和之后使用两个下划线__来用下划线替换星号。

粗体：最佳做法

使用两个星号和两个下划线都可以对内容应用粗体格式。

但是，在单词中间使用下划线可能会导致格式问题，具体取决于Markdown处理器，因此，建议使用星号。

表1.9演示了建议的粗体格式用法。

粗体和斜体

只是粗体还是斜体？如果需要，可以将两种格式结合使用以突出显示任何内容。

要同时应用粗体和斜体，请在要突出显示的文本周围添加三个星号***或三个下划线___。

如果您要强调两种格式的单词中间，请优先使用星号，在这种情况下，请使用三个星号***。

逃避主干

您可能会遇到以下情况。

参考样式Links

Markdown提供了用于创建links的另一种语法，该语法允许您更清晰地查看文档。

使用此语法，您可以将要转换为idsihablep的文本放在该段落的中间，而只需一个引用，在文档末尾，请使用与以前相同的引用添加所有links。

注意：您可以将link参考放置在文档中的任何位置，出于组织原因，我建议在最后将其引用。

由于此参考ID的创建分为两个部分，因此我们将对其进行分析。

link的第一部分

首先，我们将创建两组方括号，第一个地方是link地址的文本，也就是说，该文本将出现在页面上，第二个地方是一个标签，该标签将用作link地址的参考：

[Markdown - Wikipedia][1]
注意：位于第二对方括号内的参考link标签可以是任何东西，数字，单词，短语，甚至可以包含标点符号，只要您参考idsihABLEp即可，该文件的末尾是相同的。

例子：

[Markdown - Wikipedia][wiki_markdown.1]
[约翰·格鲁伯（John Gruber）的官方网站][JOHN GRUBER]
注意：参考标签也不区分大小写，即wiki_markdown与WIKI_MARKDOWN相同。

link的第二部分

第二部分指出了link指向的位置，即idsihABLEp的过时。

要设置link的第二部分，我们必须具备以下条件：

  1. 您之前在文档中创建的有效link参考标签，将其括在方括号中，后跟冒号和至少一个空格（例如[label]：）。
  2. O endereço do link, opcionalmente essa URL pode estar entre os sinais de menor que < e maior que >.
  3. link的标题用引号""lang-plaintext">，单引号""或括号（）表示，当然，标题是可选的。

以下所有示例对于格式化第二部分均有效：

[1]: https://en.wikipedia.org/wiki/Markdown
[1]: https://en.wikipedia.org/wiki/Markdown "Markdown - Wikipedia"
[1]: https://en.wikipedia.org/wiki/Markdown 'Markdown - Wikipedia'
[1]: https://en.wikipedia.org/wiki/Markdown (Markdown - Wikipedia)
[1]: <https://en.wikipedia.org/wiki/Markdown> "Markdown - Wikipedia"
[1]: <https://en.wikipedia.org/wiki/Markdown> 'Markdown - Wikipedia'
[1]: <https://en.wikipedia.org/wiki/Markdown> (Markdown - Wikipedia)

我再次指出，您可以将link的第二部分放在文档中的任何位置，出于组织原因，我将其添加到文件的末尾

HTML和Markdown：最佳做法

重要的是要注意，并非所有的Markdown处理器都与tags HTML兼容，在某些情况下，它们仅支持这些tags的子集。

反复试验可以是一种策略。尝试使用tags格式化内容，并检查HTML中的输出是否符合预期，或者，当然，请参阅应用程序的文档。

不要在HTML中缩进tags的行，无论是空格还是制表符都不要缩进，这会在转换时干扰元素的格式。

在块元素之前和之后插入空白行，例如: <div>, <table>, <pre>, <p>.

避免将Markdown元素放在tags HTML的内容中。
