GitHub Pages

GitHub Pages是Web的一项服务，它使用户可以托管一个由Markdown文件创建的免费网站。

  1. Markdown
  3. Markdown出版商
  5. GitHub Pages

概述

如果您已有GitHub的经验，并且需要在internet上托管网站，则可以直接使用GitHub页面存储服务。

值得一提的是，托管在GitHub页面上的页面是简单页面，例如HTML中的页面，没有服务器可以运行PHP，Python或Ruby，也没有数据库。

要托管网站，只需创建一个新的存储库，插入以Markdown格式设置的文件，然后激活GitHub页面功能。

GitHub Pages是与Markdown最兼容的工具之一，它使用静态网站生成器Jekyll作为基础。

该工具还提供用户可以选择组成其网站的主题。

当然，您可以创建一个网站并使用自己的CSS对其进行格式设置。

在下面，您可以看到Web页面，其中是GitHub页面提供的主题之一。

出于好奇，GitHub Pages和GitHub使用不同的Markdown处理器。 GitHub有其自己的Markdown处理器，而GitHub Pages使用jekyll-commonmark。

由于Markdown处理器之间的这种差异，某些格式在README.md文件中可能会以不同的方式出现。

最臭名昭著的是emoticons的使用，在GitHub的网站上将对其进行渲染，而在GitHub所生成的网站上则不会。

https://pages.github.com/是GitHub Pages的官方网站。
